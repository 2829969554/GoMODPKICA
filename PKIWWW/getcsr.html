如何为您的证书生成 CSR？

要获得证书，您需要提交有效的证书 签名请求 (CSR). 在这里你可以找到描述使用常用软件创建CSR的一些标准方法。 1.亚洲诚信第三方工具

OpenSSL

OpenSSL 是一个开源的SSL工具包， openssl 其中包括专门用于处理现代密码学各个方面的程序。 它还能够处理 RSA 密钥、X509 证书、CSR-s 和 PKCS-12 密钥库。它包含在大多数当前的 linux 发行版中， 并且是 Linux 中证书管理的基本工具。openssl

要使用 openssl 生成私钥和 CSR，请调用以下命令：

openssl req -new -nodes -keyout user-key.pem -out user.csr
系统将询问您有关证书的某些信息量。 请仔细填写，并记住，通常最依赖 CA 策略 的字段为必填项。完成后，将生成两个文件：
  • user-key.pem - 未加密的私钥（ -nodes 加密此密钥). 保护这个 钥匙！
  • user.csr - CSR 文件，并且将提交给 CA。

当您的证书被签署时 下载它 （您可以从网页获取它或等待带有它的电子邮件） CA 证书但您可能还需要一个密钥库。生成 PKCS-12 密钥库与openssl invoke命令 (假设证书在里面， user-cert.pem CA证书在里面 cacert.pem:

openssl pkcs12 -export -in user-cert.pem -out user.p12 -inkey user-key.pem -name "user" -certfile cacert.pem
在此之后，您将拥有可供使用的 PKCS-12 密钥库。

JAVA 密钥工具

随着 java 的出现，名为 的小实用程序，它是专用的 来管理密钥库。它可以在 PKCS-12 密钥库或 Java 上运行 密钥库 - JKS。本节仅引用一个新的 keytool， 它随 Java 6 一起提供。keytool 使用公钥/私钥对和自颁发的密钥库（即 对大多数应用程序无效） 证书调用：

keytool -keystore user.jks -genkeypair -alias user -keysize 2048 -keyalg rsa
现在，系统将提示您有关证书信息的信息。尽可能多地填写数据， 因为这是 CA 策略的要求（填写所有数据是安全的）。注意 that using diacretics may cause problem with 某些应用程序有效...不是全部 CA 接受此类值，因此最好仅使用 ASCII 字符。 然后使用以下命令创建 CSR：
keytool -certreq -v -keystore user.jks -alias user -file user.csr
在此之后，您将拥有以下文件：
  • user.jks - 你的密钥库
  • user.csr - 证书请求文件 提交给MODPKI CA系统
当您的证书被签名时，请下载它（您可以从网页获取它或等待带有它的电子邮件），下载 CA 证书并安装到密钥库中。 首先安装 CA 证书，然后安装您的证书（假设 CA 证书文件是：，您的证书是：）：cacert.pemuser-cert.pem

完成后，您可以检查密钥库调用：
keytool -list -v -keystore user.jks
之后，您将拥有一个用于 java 应用程序的有效密钥库。这 例如，密钥库可用于网格应用程序，如 Unicore 6 客户端 或服务器组件。
